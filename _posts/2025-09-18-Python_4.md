# 0. Introduction
{% include start-box.html class="math-box"%}
A **_string_** is a convenient data type for handling characters that are arranged in a sequence.
It is classified as one of the **_container_** data types because it can hold multiple characters. 
Among container types, it is further classified as a **_sequence_** because its elements are arranged in a specific order.
Data types classified as sequences include not only strings but also **_list_** and **_tuple_**.
In a sequence, we can use indexing and slicing.
{% include end-box.html %}

# 1. Making string literal
{% include start-box.html class="math-box"%}
{% include start-box.html class="math-box-inner" font_size="0.8em"%}
We can use both `"` and `'` to make strings.
```python
"Blue apple"
'Blue apple'
```
```python
'Blue apple'
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
To use `'` in string, we need to use `"` or `'` with `\`.
```python
"I'm Python learner."
'I\'m Python learner.'
```
```python
"I'm Python learner."
```

<br>
To use `"` in string, we need to use `'` or `"` with `\`. 
```python
"Learning \"Python\"."
'Learning "Python".'
```
```python
'Learning "Python".'
```

Like `\'` or `\"`, symbols with `\` are called **_escape character_**. 
{% include end-box.html %}

# 3. Indexing
{% include start-box.html class="math-box"%}
{% include start-box.html class="math-box-inner" font_size="0.8em"%}
`len()`: length of strings (include space)
```python
len("Blue apple")
```
```python
10
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
To get a single character from a string, we place a number representing its position, called an **_index_**, inside square brackets `[]`. 
This is called indexing.
This method can also be used on other sequences, such as `lists` and `tuples`, not just on `strings`.

```python
string = "ABCDEF"

print(string[0], string[2] , string[4])
```
```python
A C E
```
<br>
We can index last character of string with `-1`.
```python
string = "ABCDEF"

print(string[-1], string[5])
```
```python
F F
```
<br>
Negative indexing is used for reverse indexing.
```python
string = "ABCDEF"

print(string[-1], string[-2] , string[-4])
```
```python
F E C
```
{% include end-box.html %}
{% include end-box.html %}

# 4. Slicing
{% include start-box.html class="math-box"%}
The process of extending the functionality of indexing to extract a portion of a sequence is called slicing.

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
The number before the colon is the starting position.
```python
s = "ABCDEF"

s[1:]
```
```python
'BCDEF'
```
<br>
The number after the colon is the end position, which is not included.
The slice goes up to the element just before the end position.
```python
s = "ABCDEF"

s[:4]
```
```python
'ABCD'
```
Since end position is not included, we can use `len()` to include whole string.
```python
s = "ABCDEF"

s[:len(s)]

```
```python
'ABCDEF'
```
<br>
We can use start position and end position simultaneously.
```python
s = "ABCDEF"

s[2:5]
```
```python
'CDE'
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
s[:i] + s[i:] is always equal to a.
```python
s = "ABCDEF"

s[:3] + s[3:]
```
```python
'ABCDEF'
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
When we want to skip over characters while slicing, we can use a third number called the step, in addition to the start and end positions.
```python
s = "ABCDEFGH"

s[1:8:2]
```
```python
'BDFH'
```
<br>
We can call string in reverse order using negative `-1` step.
```python
s = "ABCDEFGH"

s[::-1]
```
```python
'HGFEDCBA'
```
<br>
When using a negative step, the start and end indices must also be in reverse order.
```python
s = "ABCDEFGH"

s[7:0:-2]
```
```python
'HFDB'
```
<br>
To include whole stirng in reverse oreder, we need to omit end position.
```python
s = "ABCDEFGH"
s[-1:0:-1]
```
```python
'HGFEDCB'
```
```python
s = "ABCDEFGH"
s[-1::-1]
```
```python
'HGFEDCBA'
```
{% include end-box.html %}

# 5. Immutablitily
{% include start-box.html class="math-box"%}
Python's `str` is an immutable data type. 
Since we cannot modify a part of it, we must create a new, modified object instead.

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
Since `str` is immutable data type, slicing can not change object itself.
And we cannot change a part of an immutable object.
```python
s = "ABCDEFG"

print(s[2:5])
print(s)
```
```python
CDE
ABCDEFG
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
Cretae new object using slicing.
```python
s = "ABCDEFG"
sliced_s = s[2:5]

print(s, id(s))
print(sliced_s, id(sliced_s))
```
```python
ABCDEFG 1353388688304
CDE 1353389094064
```
<br>
When a full slice is assigned to a new variable, it points to the same address.
```python
s = "ABCDEFG"
sliced_s = s[:]

print(s, id(s))
print(sliced_s, id(sliced_s))
```
```python
ABCDEFG 1353388688304
ABCDEFG 1353388688304
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
To change part of a `str`, we must create a new `str` object and reassign it to the same variable. 
When you do this, the memory address of the variable will change.
```python
s = "ABCDEFG"
print(s, id(s))

s = s[0:4] + "d" + s[5:]
print(s, id(s))
```
```python
ABCDEFG 1353388688304
ABCDdFG 1353367731696
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
We can use shorthand assignment operators with `str`.
```python
s = "ABCDEFG"
print(s, id(s))

s += " HIJ"
print(s, id(s))

s += " k"*3
print(s, id(s))
```
```python
ABCDEFG 1353388688304
ABCDEFG HIJ 1353388651184
ABCDEFG HIJ k k k 1353367797888
```
{% include end-box.html %}

# 7. Formatting
{% include start-box.html class="math-box"%}
We can adjust strings to a desired format. 
This is commonly used when printing output to the screen.

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
`%` operator
```python
print("CALL ME %s." %"ISHMAEL")
print("CALL %s %s." %("ME", "ISHMAEL")) 
```
```python
CALL ME ISHMAEL.
CALL ME ISHMAEL.
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
`str.ljust()`: Adds spaces to the right of a string to fill the remaining space up to a given width.
```python
"123456".ljust(10)
```
```python
'123456    '
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
`str.fortmat()` with positional arguments
```python
print("{} {} {}".format("White", "Red", "Blue"))
print("{2} {1} {0}".format("apple", "banana", "mandarin"))
```
```python
White Red Blue
mandarin banana apple
```
<br>
`str.fortmat()` with keyword arguments
```python
print("{a} {b} {c}".format(a = "White", b = "Red", c = "Blue"))
print("{a} {b} {c}".format(b = "apple", c = "banana", a = "mandarin"))
```
```python
White Red Blue
mandarin apple banana
```
<br>
`str.format()` with positional arguments and keyword arguments 
```python
print("{0} {a} {b}".format("Apple", a = "Banana", b = "Mandarin"))
```
```python
Apple Banana Mandarin
```
```python
print("{a} {b} {0}".format(a = "Banana", b = "Mandarin", "Apple"))
```
```python
  Cell In[95], line 1
    print("{a} {b} {0}".format(a = "Banana", b = "Mandarin", "Apple"))
                                                                    ^
SyntaxError: positional argument follows keyword argument
```
Positional argument cannot appear after keyword arguments.
<br><br>
`str.format()` with positional arguments and total length
```python
# {variable_index:total_length}
print("{0:8} | {1:5}".format("First", "Second"))
```
```python
First    | Second
```
<br>
`str.format()` with keyword arguments total length 
```python
# {variable_index:total_length}
print("{a:8} | {b:5}".format(a = "First", b = "Second"))
```
```python
First    | Second
```
<br>
`str.format()` with location in total length
```python
# <, ^, > : left, center, right
#{variable_index:location total_length}

print("{0:<10} | {1:^10} | {2:>10}".format("Left", "Center", "Right"))
```
```python
Left       |   Center   |      Right
```
<br>
`str.format()` with padding (pad empty space)
```python
#{variable_index:padding_character location total_length}
print("{0:1<8} | {1:2^8} | {2:3>8}".format("Left", "Center", "Right"))
```
```python
Left1111 | 2Center2 | 333Right
```

# 8. F-string method
{% include start-box.html class="math-box"%}
{% include start-box.html class="math-box-inner" font_size="0.8em"%}
Prefix a string literal with an `f` or `F` and then write expressions inside `{}`.
```python
someone = "ME"

print(f"CALL {someone} ISHMAEL.")
```
```python
CALL ME ISHMAEL.
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
F-string operation
```python
print(f"Addition {7+3}")
```
```python
Addition 10
```
<br>
```python
a = 7
b = 3

print(f"Addition: {a} + {b} = {a+b}")
```
```python
Addition: 7 + 3 = 10
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
F-string with total length
```python
num = "123456"

f"{num:10}"
```
```python
'123456    '
```
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
`float` expression
```python
num = 123.56789
print(f"Result:{num:15.6}")
print(f"Result:{num:15.6f}")
```
```python
Result:        123.568
Result:     123.567890
```
In an f-string, when you specify precision without the `f` type code, it sets the total number of significant digits. 
When we use it with the `f` type code, it sets the number of digits after the decimal point.
{% include end-box.html %}

{% include start-box.html class="math-box-inner" font_size="0.8em"%}
F-string alignment
{% include end-box.html %}
```python
num = 1.23456789
print(str(num))
print(f"Result: {num:<{15}.{5}}")
print(f"Result: {num:>{15}.{5}}")
print(f"Result: {num:^{15}.{5}}")
```
```python
1.23456789
Result: 1.2346         
Result:          1.2346
Result:     1.2346
```
